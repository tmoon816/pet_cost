# 产品愿景（vision.md）

本文件是 PM cron agent 的锚点。它会基于本文件提产品提案。修改后直接 push 即可，不需要 spec 流程。

---

## 1. 产品定位

面向**个人经营的小型宠物店**（1 人前台 + 经营），帮老板省掉手工记账的时间，沉淀客户和宠物档案，并通过消费历史回答「这个客户是不是老客 / 是不是值得维护」这类问题。**不做**线上电商、客户端 App、在线支付、多门店 SaaS、AI 推荐与健康预测。

---

## 2. 核心用户和场景

1. **接待时**：客户进店，老板能 5 秒内查到「上次什么时候来 / 都消费过什么 / 累计花了多少」，决定要不要给老客优惠或主动推荐
2. **打烊前**：老板花 5 分钟而不是 30 分钟把今天的消费录入系统，不再事后翻小票对账
3. **月底**：老板想看本月「新客数 vs 回头客数」「哪些老客最近 3 个月没来了（流失预警）」「Top 10 高价值客户」
4. **维护客情**：老板能从客户档案点开看完整的消费历史和宠物名下的所有记录，不用脑子记

---

## 3. 现阶段不做什么

- ❌ 线上电商 / 客户端 App / 任何对外销售页面或购物车
- ❌ 在线支付集成（微信/支付宝），仍走线下收款
- ❌ 多门店 / 多租户 / SaaS 化
- ❌ AI 推荐 / 健康预测 / 智能客服等炫但难落地的能力
- ❌ 复杂的会员积分 / 等级 / 优惠券体系（v1 不做）
- ❌ 员工业绩 / 服务订单 / 提成结算（个人小店没这个需求）

---

## 4. 当前阶段

v1 重点是把现有 5 模块（客户 / 宠物 / 消费 / 分类 / 预算）的 CRUD 体验打磨稳，让一个真实小店能真正用起来。围绕「客户回头率不清晰」这个核心痛点，做小增量：客户消费历史聚合、回头客标签、流失预警卡片。其他大方向（员工、报表深化、移动端）暂缓。
