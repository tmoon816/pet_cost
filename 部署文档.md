# 宠物成本管理系统 部署文档
## 📋 项目概述
本系统是面向宠物医院/宠物店的前后端分离的成本管理系统，实现了客户→宠物→消费记录三级数据管理，包含数据大盘、账单管理、宠物档案、消费分类管理、月度预算管理五大核心模块。
## 🛠️ 技术栈
- **前端**：Vue 3 + Vite + Element Plus + ECharts
- **后端**：Python 3.11+ + FastAPI + SQLAlchemy 2.x + Alembic
- **开发数据库**：SQLite
- **生产数据库**：MySQL 8.0+
## 📦 本地部署步骤
### 1. 环境准备
确保本地已安装：
- Python 3.11+
- Node.js 18+
- uv（Python包管理器）
### 2. 克隆项目
```bash
git clone <仓库地址>
cd pet_cost
```
### 3. 后端部署
```bash
# 进入后端目录
cd backend
# 安装依赖
uv sync
# 复制配置文件
cp .env.example .env
# 修改配置文件（可选，根据需要修改数据库连接等）
vi .env
# 执行数据库迁移
uv run alembic upgrade head
# 导入初始分类数据（可选）
uv run python -m app.seed
# 启动后端服务
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
后端启动后，访问 http://localhost:8000/docs 可以查看Swagger接口文档。
### 4. 前端部署
```bash
# 进入前端目录
cd frontend
# 安装依赖
npm install
# 复制环境配置文件
cp .env.development .env.local
# 修改配置文件（可选，后端地址如果不是默认的话需要修改）
vi .env.local
# 启动前端开发服务
npm run dev
```
前端启动后，访问 http://localhost:3000 即可使用系统。
### 5. 生产部署
```bash
# 前端编译
cd frontend
npm run build
# 编译后的静态文件在 dist 目录，可以使用 Nginx 部署
# 后端可以使用 gunicorn/uvicorn 部署
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```
## 🔧 MySQL适配说明
本系统已经完全兼容MySQL 8.0+，切换到MySQL只需要修改后端`.env`配置文件：
```env
# SQLite配置（开发用）
# DATABASE_URL=sqlite:///./pet_cost.db
# MySQL配置（生产用）
DATABASE_URL=mysql+pymysql://<用户名>:<密码>@<地址>:<端口>/<数据库名>?charset=utf8mb4
```
然后重新执行迁移即可：
```bash
uv run alembic upgrade head
```
## ✅ 功能测试方法
### 核心功能测试
1. **客户管理**：新增/编辑/删除客户，查看客户列表
2. **宠物管理**：新增/编辑/删除宠物，关联客户，查看宠物档案
3. **账单管理**：新增/编辑/删除消费记录，关联宠物和分类，多条件筛选
4. **分类管理**：新增/编辑/删除消费分类，启用/禁用分类
5. **预算管理**：设置全局/宠物/分类月度预算，查看预算使用情况，超支提醒
6. **数据大盘**：查看消费统计、分类占比、宠物消费排行、月度消费趋势
### 测试账号
v1版本不需要登录，直接访问即可使用。
